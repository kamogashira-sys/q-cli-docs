## 最新バージョン

### v1.25.0 CLI（2026-02-04）

**主要な変更**:
- 🔌 **Agent Client Protocol (ACP)**: JetBrains IDEs、Zed等のACP対応エディタでKiroをカスタムエージェントとして使用可能
  - `kiro-cli acp`コマンドでJSON-RPC over stdin/stdout通信

  - セッション保存: `~/.kiro/sessions/cli/`

  - `/help`コマンドで切り替え、`/help <質問>`で直接質問
  - コマンド、ツール、設定、機能について即座に回答

  - Globパターン（`test-*`等）をサポート
  
- 🚀 **Exit Codes for CI/CD**: 構造化された終了コードで自動化をサポート
  - コード0: 成功、コード1: 一般的な失敗、コード3: MCPサーバー起動失敗
  - `--require-mcp-startup`オプションでMCPサーバー必須化

## Subagent Access Control（v1.25.0）

### 概要
v1.25.0で追加されたサブエージェントの細粒度アクセス制御機能。

### availableAgents設定

**目的**: サブエージェントとして起動可能なエージェントを制限

**設定例**:
```json
{
  "toolsSettings": {
    "subagent": {
      "availableAgents": ["reviewer", "tester", "docs-*"]
    }
  }
}
```

**動作**: `reviewer`、`tester`、`docs-*`パターンに一致するエージェントのみ起動可能

### trustedAgents設定

**目的**: 承認プロンプトなしで実行可能なエージェントを指定

**設定例**:
```json
{
  "toolsSettings": {
    "subagent": {
      "trustedAgents": ["reviewer", "tester"]
    }
  }
}
```

**動作**: `reviewer`、`tester`は承認プロンプトなしで即座に起動

### 両設定の組み合わせ

**設定例**:
```json
{
  "toolsSettings": {
    "subagent": {
      "availableAgents": ["reviewer", "tester", "analyzer", "docs-*"],
      "trustedAgents": ["reviewer", "tester"]
    }
  }
}
```

**動作**: 
- 4種類のエージェントが起動可能
- `reviewer`、`tester`は承認不要
- `analyzer`、`docs-*`は承認プロンプト表示

### Globパターンサポート

**対応パターン**:
- `test-*`: `test-`で始まる全エージェント
- `*-agent`: `-agent`で終わる全エージェント
- `dev-*-test`: `dev-`で始まり`-test`で終わる全エージェント

### ユースケース

#### オーケストレーターエージェント
複数の専門エージェントを統括するエージェント設定例。

#### チーム開発
承認済みエージェントのみを使用する制限環境。

### ベストプラクティス

1. **最小権限の原則**: 必要なエージェントのみを許可
2. **信頼エージェントの慎重な選択**: 自動実行されるため慎重に
3. **Globパターンの活用**: 命名規則を統一してパターンで管理

**出典**: [Subagents - Configuring Subagent Access](https://kiro.dev/docs/cli/chat/subagents/#configuring-subagent-access)
```

**配置**: 既存の「サブエージェントの種類」セクションの後

## Enterprise Web Tools Governance（v1.25.0）

### 概要
v1.25.0で追加された組織全体でのWeb Tools制御機能。管理者が`web_search`と`web_fetch`ツールを組織レベルで無効化可能。

### 機能詳細

#### 管理者による制御
- Kiroコンソールから組織全体の設定を管理
- `web_search`と`web_fetch`ツールを一括無効化
- ユーザーは`/tools`コマンドで無効化通知を確認

#### 設定方法

**手順**:
1. Kiroコンソールを開く
2. **Settings**を選択
3. **Shared settings**配下の**Web search and web fetch tools**を**Off**に切り替え

#### ユーザー側の動作

**無効化時**:
- `/tools`コマンドで通知表示: "Web access has been disabled by your administrator"
- `web_search`、`web_fetch`ツールは利用不可
- エージェントはWeb検索・取得を実行できない

### ユースケース

#### セキュリティポリシー遵守
外部Webアクセスを制限する組織のポリシー対応。

#### コンプライアンス要件
特定の業界規制でWeb接続を制限する必要がある場合。

#### 開発環境の制御
本番環境でのWeb接続を禁止し、ローカルリソースのみ使用。

### 個別URL制御との関係

- **Enterprise設定**: 組織全体でWeb Tools自体を無効化
- **Granular URL Permissions**: 個別ユーザーがURL単位で制御

**優先順位**: Enterprise設定 > Granular URL Permissions

### ベストプラクティス

1. **段階的な導入**: まず個別制御で運用、必要に応じて組織全体制御
2. **ユーザーへの周知**: 無効化前にユーザーに通知
3. **代替手段の提供**: 必要な情報へのアクセス方法を別途提供

**出典**: [Enterprise Settings - Web Tools](https://kiro.dev/docs/cli/enterprise/settings/#web-tools)
```

**配置**: 既存の「ベストプラクティス」セクションの前
